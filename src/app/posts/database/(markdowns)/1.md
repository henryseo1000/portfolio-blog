# Apache Tomcat 설치하기

## 개요

이번에 `Spring`을 공부하고 간단한 프로젝트를 하게 되면서, 

`Spring` 서버를 실행시키기 위해 `Apache Tomcat`이 필요하게 되어 이렇게 설치하게 되었습니다.

제가 정리하려고 하는 방법은 크게 2가지 입니다.

1. 맥북에 설치하여 사용하기
2. `Raspberry PI`에 설치해 사용해보기

## Raspberry PI에서 설치하기

이번에는, 라즈베리 파이에 설치해보겠습니다.

먼저, `java`를 설치해주어야 합니다.

그 전에, `java`의 버전 리스트를 받아와봅시다.

```bash showLineNumbers
sudo apt-get update

sudo apt list openjdk*

#또는 특정 버전에 대한 리스트만 받아오고 싶을 경우
sudo apt list openjdk-[버전]*
```

저는 설치하고 싶은 `java` 버전을 17로 정했기 때문에,

아래와 같이 명령어를 입력하였습니다.

```bash
sudo apt-get install openjdk-17-jdk
```

오류가 나지 않았다면, 아래와 같이 입력하여 `java`와 `javac` 명령어가 잘 먹는지 확인해봅시다.

```bash
java --version
javac -version
```

![image.png](1/image.png)

위 맥북의 과정과 같은 과정을 거치되, 위 `tar.gz` 파일을 설치하는 부분에서 링크 복사를 합니다.

![image.png](1/image%201.png)

이제 이 부분을 `wget` 명령어로 설치해주면 됩니다.

아래와 같이 설치해줍니다.

```bash
wget [복사한 링크 주소]

wget https://dlcdn.apache.org/tomcat/tomcat-10/v10.1.28/bin/apache-tomcat-10.1.28.tar.gz
```

![스크린샷 2024-08-11 오후 8.35.59.png](1/%25E1%2584%2589%25E1%2585%25B3%25E1%2584%258F%25E1%2585%25B3%25E1%2584%2585%25E1%2585%25B5%25E1%2586%25AB%25E1%2584%2589%25E1%2585%25A3%25E1%2586%25BA_2024-08-11_%25E1%2584%258B%25E1%2585%25A9%25E1%2584%2592%25E1%2585%25AE_8.35.59.png)

홈 디렉토리에 잘 설치가 된 것을 볼 수 있습니다.

이제 해당 파일의 압축을 풀어주겠습니다.

아래 명령어를 입력해 압축을 풀어봅시다.

```bash
sudo tar xvf [받은 tar.gz 파일 이름] -C /usr/local
```

위 명령어는 `/usr/local`에 압축을 풀겠다는 의미입니다.

따라서 이제 `/usr/local`에 압축이 잘 풀렸는지 확인하겠습니다.

![스크린샷 2024-08-11 오후 8.40.19.png](1/%25E1%2584%2589%25E1%2585%25B3%25E1%2584%258F%25E1%2585%25B3%25E1%2584%2585%25E1%2585%25B5%25E1%2586%25AB%25E1%2584%2589%25E1%2585%25A3%25E1%2586%25BA_2024-08-11_%25E1%2584%258B%25E1%2585%25A9%25E1%2584%2592%25E1%2585%25AE_8.40.19.png)

하위 폴더에 `apache-tomcat` 폴더가 잘 생성된 걸 확인할 수 있습니다.

이제 `cd` 명령어를 통해 아래 `apache-tomcat` 폴더로 이동하겠습니다.

```bash
cd apache-tomcat-10.1.28
```

현재는 `apache-tomcat`의 압축만 풀었을 뿐, 시스템의 서비스로서 등록하지는 않았습니다.

이제부터 해당 과정을 진행해보겠습니다.

먼저 `tomcat.service` 파일을 생성합니다.

```bash
sudo vi /etc/systemd/system/tomcat.service
```

위 명령어로 파일이 생성되었다면, 아래 내용을 입력해주면 됩니다.

```bash
[Unit]
Description=tomcat 10 #톰캣 버전에 맞추어 입력
After=network.target syslog.target

[Service]
Type=forking
Environment="JAVA_HOME=자바 경로"
Environment="CATALINA_HOME=톰캣 경로"
User=root
Group=root
ExecStart=톰캣경로/bin/startup.sh
ExecStop=톰캣경로/bin/shutdown.sh

[Install]
WantedBy=multi-user.target
```

모두 적어주고 나서는 `esc` 키를 누르고 저장합시다.

저는 아래와 같이 다른 블로그를 참고하여 새로 작성해주었습니다.

![스크린샷 2024-08-11 오후 9.33.51.png](1/%25E1%2584%2589%25E1%2585%25B3%25E1%2584%258F%25E1%2585%25B3%25E1%2584%2585%25E1%2585%25B5%25E1%2586%25AB%25E1%2584%2589%25E1%2585%25A3%25E1%2586%25BA_2024-08-11_%25E1%2584%258B%25E1%2585%25A9%25E1%2584%2592%25E1%2585%25AE_9.33.51.png)

위와 같이 서비스와 관련된 `.service`를 설정해줄 경우에는 아래와 같은 규칙을 따릅니다.

1. `[]`의 대괄호로 구분되는 각 부분은 `Section`이라고 하는데, 각 `Section`의 이름 `(Unit, Service)`은 한 줄만 쓰고, 각 `Section`에 해당하는 내용은 줄바꿈으로 구분합니다.
2. 각 서비스의 `Section`의 이름은 대소문자를 구분하며, 잘못된 `Section`의 이름을 사용할 경우 오류가 발생합니다.

제가 이걸 왜 아냐구요?

`Unit` 대문자로 썼다가 톰캣이 실행 안된다고 저항하는 바람에… 삽질 좀 했습니다.

또한 한 가지 더 :

`java` 경로를 작성할 경우에는 자바 경로 그대로 나오는걸 쓰지 말고 끝에 `bin/java` 부분은 떼고 씁니다.

`JDK`의 경로를 지정해 주는거지, `JRE`가 어디 있는지는 궁금하지 않다고 하네요… **~~이 요망한 녀석…~~**

![스크린샷 2024-08-11 오후 9.31.25.png](1/%25E1%2584%2589%25E1%2585%25B3%25E1%2584%258F%25E1%2585%25B3%25E1%2584%2585%25E1%2585%25B5%25E1%2586%25AB%25E1%2584%2589%25E1%2585%25A3%25E1%2586%25BA_2024-08-11_%25E1%2584%258B%25E1%2585%25A9%25E1%2584%2592%25E1%2585%25AE_9.31.25.png)

이렇게 잘 설정해주고 나면, 아래의 명령어를 순서대로 입력해줍니다.

```bash
systemctl enable tomcat.service

# start를 했을 때 에러가 뜨면 안됩니다. 에러가 뜬다면 어딘가 설정을 잘못하신 겁니다.
systemctl start tomcat.service

# start 시 에러가 뜨지 않는다면 아래 명령어 입력해 확인
systemctl status tomcat.service
```

![스크린샷 2024-08-11 오후 9.41.14.png](1/%25E1%2584%2589%25E1%2585%25B3%25E1%2584%258F%25E1%2585%25B3%25E1%2584%2585%25E1%2585%25B5%25E1%2586%25AB%25E1%2584%2589%25E1%2585%25A3%25E1%2586%25BA_2024-08-11_%25E1%2584%258B%25E1%2585%25A9%25E1%2584%2592%25E1%2585%25AE_9.41.14.png)

위와 같이 초록색으로 **active(running)**이 뜨면 성공입니다.

만약 성공하지 못하셨다면 아래 명령어를 입력 후 :

```bash
systemctl disable tomcat.service
```

`tomcat.service`를 수정하신 뒤 다시 위 명령어를 차례대로 입력해주세요.

그렇지 않으면 제대로 반영이 되지 않으니 주의해주셔야 합니다.

이상으로 라즈베리 파이에 `apache-tomcat`을 설치하고 설정하는 방법을 알아보았습니다.
